| 상태 | 신뢰도 | 설명 | 처리 방식 |

| **CONFLICT** | 70% | 데이터 충돌 | 수동 검토 |

### 6.4 Circuit Breaker 패턴

```javascript
// Circuit Breaker 상태
const STATES = {
  CLOSED: 'closed',      // 정상 작동
  OPEN: 'open',          // 차단 (요청 즉시 실패)
  HALF_OPEN: 'half-open' // 테스트 모드
};

// 설정
const config = {
  failureThreshold: 5,    // 5회 실패 시 OPEN
  successThreshold: 3,    // 3회 성공 시 CLOSED
  timeout: 30000,         // 30초 후 HALF_OPEN
  retryDelay: [1000, 2000, 4000, 8000, 16000]  // 지수 백오프
};
```

### 10.3 핵심 교훈

1. **공식 API 우선**: 크롤링보다 API가 항상 안정적이고 법적으로 안전
2. **Dual-Source 필수**: 단일 소스 의존은 위험
3. **AI Vision은 Fallback**: 주요 데이터 수집 방식으로 사용하지 말 것
4. **pykrx 회피**: IP 차단 위험이 너무 높음
5. **법적 검토 필수**: 특히 네이버 금융 크롤링은 피할 것
6. **2026 규제 대비**: AI 기본법, 데이터 보호법 준수 계획 수립
